2、请求方式

GET

3、输入参数

无

4、返回结果

1、请求地址

http://192.168.0.105:8076/api/auth/v1/authentication

2、请求方式

POST(application/json)

3、输入参数

输入示例：{ "authCode": "7f3f8e484abf4363b7f35471e3a8cfae", "accessKey": "3c08f763a35641d398ed54ad16d6140c", "signature": "f5907599f202bb042c7ea55af67ade8440423f944833deec3d59dc7345f5ab3d" }

键 类型 描述 示例值
authCode String 授权码， 7f3f8e484abf4363b7f35471e3a8cfae
accessKey String `api访问ak`，登录系统后在密钥凭证功能里可获得 3c08f763a35641d398ed54ad16d6140c
signature String 签名，使用authCode作为key，对`api访问ak密钥`进行sm3-hmac运算得到的结果，格式为hex 0

4、返回结果

返回示例：{"code":0, "msg": "操作成功！","data": "tinyurl_7552f531fc0d4b6e9547178aec362e41"}

键 类型 描述 示例值
code String 标识码，0为成功，其他均为失败 0
data String 会话token，后续接口均须在请求header里带此token进行访问（键为`token`），会话时长为30分钟，操作时自动刷新会话时间 0
